Самое сложное техническое решение

Ролевая модель доступа + оптимизация статистики (batch-запросы)

Проблема

Нужно разграничить права 3 ролей и при этом быстро строить сводки по отделам без сотен SQL-запросов (N+1).

Решение

Проверки can_create / can_deactivate в сервисах. Агрегация stats через GROUP BY и joinedload. Переиспользование employee_stats.

Результат

Админ видит всю клинику, руководитель — только свой отдел. Дашборд загружается за 1–3 запроса вместо 15+.
